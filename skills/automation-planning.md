# Automation Planning Skill

## Purpose
Design dependable automations before implementing them.

## Use When
Automating business processes, API workflows, notifications, reports, scheduled jobs, or no-code flows.

## Inputs
- Trigger.
- Inputs and outputs.
- Systems involved.
- Approval and failure rules.

## Method
1. Map the manual process.
2. Identify decision points and exceptions.
3. Choose tool path.
4. Define idempotency, retries, and logs.
5. Test with safe sample data.

## Output
- Workflow diagram or outline.
- Tool recommendation.
- Implementation steps.
- Error handling.
- Test plan.
