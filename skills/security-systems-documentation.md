# Security Systems Documentation Skill

## Purpose
Create safe, operator-ready documentation for security systems.

## Use When
Documenting camera systems, access control, alarms, intercoms, retention, maintenance, or incident procedures.

## Inputs
- Site and system scope.
- Device inventory.
- User roles and retention needs.
- Vendor platform details.

## Method
1. Define audience and sensitivity.
2. Inventory devices and dependencies.
3. Document procedures with verification.
4. Redact sensitive details not needed by the reader.
5. Add maintenance and escalation steps.

## Output
- System overview.
- Inventory.
- Procedures.
- Maintenance schedule.
- Escalation contacts or roles.
