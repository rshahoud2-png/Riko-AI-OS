# Vercel Deployment Skill

## Purpose
Deploy and troubleshoot Vercel-hosted applications.

## Use When
Configuring Vercel, fixing builds, setting env vars, adding domains, or optimizing previews.

## Inputs
- Framework, package manager, and build command.
- Logs and deployment URL.
- Environment variable names only.
- Domain requirements.

## Method
1. Reproduce build locally when possible.
2. Check framework detection and output settings.
3. Verify env var availability by environment.
4. Inspect routing, functions, and runtime limits.
5. Validate preview and production behavior.

## Output
- Diagnosis.
- Config changes.
- Env checklist.
- Verification.
- Rollback.
