# GitHub Automation Skill

## Purpose
Use GitHub to organize personal repositories and automate repeatable maintenance.

## Use When
Creating repo structure, issues, PR templates, labels, release notes, GitHub Actions plans, project hygiene, or automation checklists.

## Inputs
- Repository and objective.
- Current branch, issue, PR, or file context.
- Desired automation or maintenance outcome.
- Safety limits.

## Method
1. Inspect repository state through GitHub tools.
2. Identify the smallest useful automation or cleanup.
3. Update files/issues/PRs with clear names and descriptions.
4. Avoid secrets in workflows or examples.
5. Verify changed files or GitHub objects.

## Output
- Action summary.
- Files/issues/PRs changed.
- Verification.
- Follow-up tasks.
