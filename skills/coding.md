# Coding Skill

## Purpose
Implement focused, maintainable code changes.

## Use When
Building features, fixing bugs, refactoring, or wiring integrations.

## Inputs
- Requirements and acceptance criteria.
- Repository context.
- Existing commands.
- Test expectations.

## Method
1. Inspect existing patterns.
2. Plan the smallest coherent change.
3. Edit locally and keep scope tight.
4. Add or update tests when risk warrants.
5. Run checks and summarize.

## Output
- Files changed.
- Behavior changed.
- Tests run.
- Residual risk.
