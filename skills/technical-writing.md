# Technical Writing Skill

## Purpose
Produce clear technical documents that help a reader take action.

## Use When
Writing README files, runbooks, SOPs, API docs, admin guides, or customer handoffs.

## Inputs
- Audience.
- Purpose.
- Source facts.
- Required format.

## Method
1. Define reader job.
2. Create a scannable structure.
3. Put prerequisites before steps.
4. Make each step verifiable.
5. Add troubleshooting and ownership.

## Output
- Draft document.
- Assumptions.
- Verification checklist.
- Maintenance notes.
