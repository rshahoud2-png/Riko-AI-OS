# Root Cause Analysis Skill

## Purpose
Find the underlying cause of an incident and prevent recurrence.

## Use When
An outage, defect, process failure, or repeated issue needs more than a quick fix.

## Inputs
- Timeline.
- Impact.
- Detection method.
- Logs and artifacts.
- Changes before incident.

## Method
1. Build a factual timeline.
2. Separate trigger, contributing factors, and root cause.
3. Identify detection and response gaps.
4. Define corrective and preventive actions.
5. Assign owners and due dates.

## Output
- Executive summary.
- Timeline.
- Root cause and contributing factors.
- Customer/business impact.
- Action items.
