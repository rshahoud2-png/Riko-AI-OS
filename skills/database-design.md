# Database Design Skill

## Purpose
Design clear, secure, performant data models.

## Use When
Creating schemas, migrations, RLS policies, indexes, or data access patterns.

## Inputs
- Entities and relationships.
- Query patterns.
- Access roles.
- Data retention and compliance needs.

## Method
1. Model entities and cardinality.
2. Define constraints and indexes.
3. Design access control with least privilege.
4. Plan migrations and backfills.
5. Validate with representative queries.

## Output
- Schema design.
- Access rules.
- Migration plan.
- Performance notes.
- Validation plan.
