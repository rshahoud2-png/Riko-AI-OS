# Network Diagnostics Skill

## Purpose
Diagnose network issues from physical link through application reachability.

## Use When
Troubleshooting LAN, WAN, VPN, Wi-Fi, DNS, DHCP, firewall, packet loss, or latency.

## Inputs
- Topology.
- IP/VLAN/DNS/DHCP details.
- Symptoms and affected users.
- Logs and test results.

## Method
1. Confirm scope and timeline.
2. Test local link and addressing.
3. Test gateway, DNS, routing, and firewall.
4. Test application endpoint.
5. Document root cause and final config.

## Output
- Scope.
- Evidence.
- Likely fault domain.
- Fix or next tests.
- Rollback and monitoring notes.
