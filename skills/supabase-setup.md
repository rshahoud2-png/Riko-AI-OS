# Supabase Setup Skill

## Purpose
Set up Supabase projects with secure schema, auth, and access policies.

## Use When
Starting or repairing Supabase Auth, Postgres, Storage, Edge Functions, or realtime.

## Inputs
- Project purpose.
- User roles.
- Data model.
- Client/server architecture.

## Method
1. Define schema and roles together.
2. Enable RLS for protected tables.
3. Write narrow policies.
4. Keep service-role operations server-side.
5. Test with anon, authenticated, and privileged paths.

## Output
- Project checklist.
- SQL/policy plan.
- Client integration notes.
- Security checks.
- Test cases.
