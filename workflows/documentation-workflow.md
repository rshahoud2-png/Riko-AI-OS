# Documentation Workflow

1. Define audience and purpose.
2. Gather source truth.
3. Choose document type: README, SOP, runbook, proposal, guide, or reference.
4. Draft from overview to action.
5. Add verification, troubleshooting, and ownership.
6. Review for accuracy and sensitive information.

## Output

- Document.
- Assumptions.
- Maintenance owner.
- Next review date.
