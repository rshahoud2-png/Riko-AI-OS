# Coding Workflow

1. Inspect existing code and commands.
2. Identify the smallest coherent change.
3. Implement using project patterns.
4. Add or update focused tests.
5. Run checks.
6. Review diff for regressions.
7. Summarize changed files and verification.

## Gates

- Requirements clear.
- Tests pass or limitation documented.
- No secrets added.
- Code reviewer pass completed for risky changes.
